## 简介
本指南面向首次使用"基金账户诊断技能"的用户，帮助你在30分钟内完成环境准备、安装依赖、准备数据并成功运行第一个诊断报告。你将学会：
- 如何准备Python 3.8+环境与依赖库
- 如何准备两种数据输入方式：基金代码列表与交易记录Excel文件
- 如何运行基础诊断命令、生成HTML可视化报告、查看交易统计与指定模块诊断
- 如何配置MCP数据源与API密钥

## 环境准备

### Python版本
- Python 3.8及以上

### 依赖安装
```bash
pip3 install pandas>=1.3.0 numpy>=1.20.0 empyrical>=0.5.5
```

或使用 requirements.txt：
```bash
pip3 install -r requirements.txt
```

### 可选依赖
- `coze_workload_identity`：用于HTTP请求（Coze平台已预装），缺失时自动回退到urllib标准库

## 数据输入方式

### 方式一：基金代码列表
适用场景：已有明确持仓的基金代码清单
```bash
python scripts/diagnostic_report.py --funds 000001,000002,000003
```
数据来源：MCP qieman服务器（当可用时），否则使用模拟数据

### 方式二：交易记录Excel文件（推荐）
适用场景：希望从历史交易明细生成诊断报告
```bash
python scripts/diagnostic_report.py --transaction-file ./transactions.xlsx
```

#### Excel列名映射（自动识别）
| 字段 | 支持的列名 |
|------|----------|
| 基金代码 | 基金代码、代码 |
| 基金名称 | 基金名称、基金简称 |
| 业务名称 | 业务名称、操作、交易类型 |
| 申请金额 | 申请金额 |
| 确认金额 | 确认金额 |
| 确认份额 | 确认份额 |
| 单位净值 | 产品单位净值、净值、单位净值 |

#### 支持的业务类型
申购、赎回、分红、定期定额申购、基金转换、转入、转出

#### 解析逻辑要点
- 自动识别列名（精确匹配→模糊匹配）
- 金额字段支持带逗号格式（如"5,000.00"）
- 净值为空时默认为1.0
- 份额低于1e-6视为已清仓
- 基金转换自动处理双端（转出+转入）

## 基础使用示例

### 示例1：从交易记录生成诊断报告
```bash
python scripts/diagnostic_report.py --transaction-file ./transactions.xlsx
```

### 示例2：快速诊断基金账户
```bash
python scripts/diagnostic_report.py --funds 000001,000002,000003
```

### 示例3：显示交易统计
```bash
python scripts/diagnostic_report.py --transaction-file ./transactions.xlsx --show-stats
```

### 示例4：指定模块诊断
```bash
python scripts/diagnostic_report.py --funds 000001,000002 --modules overview,performance,risk
```

### 示例5：保存报告到文件
```bash
python scripts/diagnostic_report.py --funds 000001,000002 --output diagnostic_report.json
```

### 示例6：生成HTML可视化报告
```bash
# 方式1: 一步到位
python scripts/diagnostic_report.py --funds 000001,000002 --format html --output diagnostic_report.html

# 方式2: 从已有JSON报告生成HTML
python scripts/generate_html_report.py --input diagnostic_report.json --output diagnostic_report.html
```

**HTML报告特性**：
- ECharts 5交互式图表（13种图表类型）
- 品牌色 #0052D9，红涨绿跌
- 响应式布局，自包含HTML文件

## MCP数据源配置

### 数据源信息
- 服务器：qieman MCP（https://stargate.yingmi.com/mcp/v2）
- 协议：JSON-RPC 2.0
- 认证：x-api-key header

### 凭证配置
- 环境变量：`COZE_QIEMAN_API_{SKILL_ID}`
- 若未配置，脚本将使用内置默认API Key

### 降级策略
当API不可用时，自动使用基于基金代码哈希的确定性模拟数据

## 命令行参数速查

| 参数 | 说明 |
|------|------|
| `--funds` | 基金代码，逗号分隔 |
| `--transaction-file` | 交易记录Excel文件路径 |
| `--modules` | 分析模块（diagnosis/overview/performance/risk/allocation/correlation/evaluation/rebalance/summary） |
| `--output` | 输出文件路径 |
| `--format` | 输出格式：json 或 html |
| `--show-stats` | 显示交易记录统计摘要 |

### 可用模块
- `diagnosis` — 账户诊断总览（综合得分、等级、配置偏离度）
- `overview` — 持仓概览（基金数量、总市值、盈亏、集中度预警）
- `performance` — 收益风险表现（累计收益、夏普比率、最大回撤）
- `risk` — 风险提示（情景分析、市场风险、流动性风险）
- `allocation` — 组合配置诊断（资产配置、行业穿透、重仓股）
- `correlation` — 相关性分析（相关系数矩阵、高相关对）
- `evaluation` — 单只基金评价（主动型/指数型双轨评价）
- `rebalance` — 调仓建议（超配/低配、替换建议、批次安排）
- `summary` — 报告总结（核心发现、关键风险、优化建议）
